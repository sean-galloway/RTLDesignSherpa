# AXI4 Slave Interface

The controller exposes a standard AXI4 slave for SoC connectivity.

## Channels

Standard AXI4 — all five channels (AW, W, B, AR, R) are mandatory. No exclusive access support; `AxLOCK` is ignored.

### AW (Write Address) — SoC master → controller slave

| Signal      | Width                   | Notes                                     |
|-------------|-------------------------|-------------------------------------------|
| `awid`      | `AXI_ID_WIDTH`          | Transaction ID                            |
| `awaddr`    | `AXI_ADDR_WIDTH`        | Flat byte address                         |
| `awlen`     | 8                       | Burst length minus 1                      |
| `awsize`    | 3                       | Beat size; up to bus width                |
| `awburst`   | 2                       | `INCR` mandatory; `FIXED`/`WRAP` optional |
| `awcache`   | 4                       | Ignored; treated as `0010` (modifiable)  |
| `awprot`    | 3                       | Ignored                                   |
| `awqos`     | 4                       | Forwarded to scheduler as priority hint   |
| `awvalid`   | 1                       | Master asserts                            |
| `awready`   | 1                       | Controller asserts when slot available    |

### W (Write Data)

| Signal      | Width                                  | Notes                          |
|-------------|----------------------------------------|--------------------------------|
| `wdata`     | `AXI_DATA_WIDTH`                       | Beat data                      |
| `wstrb`     | `AXI_DATA_WIDTH/8`                     | Byte enables; inverted to wrdata_mask |
| `wlast`     | 1                                      | Final beat of burst            |
| `wvalid`    | 1                                      | Master asserts                 |
| `wready`    | 1                                      | Controller asserts on FIFO space |

Note: AXI4 W channel has no `wid`; beats are presumed in the order their AW transactions were issued. The controller tracks IDs internally based on AW arrival order.

### B (Write Response)

| Signal      | Width                  | Notes                                |
|-------------|------------------------|--------------------------------------|
| `bid`       | `AXI_ID_WIDTH`         | Matches `awid`                       |
| `bresp`     | 2                      | `OKAY` or `SLVERR`                   |
| `bvalid`    | 1                      | Controller asserts on response ready |
| `bready`    | 1                      | Master asserts                       |

### AR (Read Address)

| Signal      | Width                  | Notes                                |
|-------------|------------------------|--------------------------------------|
| `arid`      | `AXI_ID_WIDTH`         | Transaction ID                       |
| `araddr`    | `AXI_ADDR_WIDTH`       | Flat byte address                    |
| `arlen`     | 8                      | Burst length minus 1                 |
| `arsize`    | 3                      | Beat size                            |
| `arburst`   | 2                      | `INCR` mandatory; others optional    |
| `arcache`   | 4                      | Ignored                              |
| `arprot`    | 3                      | Ignored                              |
| `arqos`     | 4                      | Forwarded to scheduler as priority   |
| `arvalid`   | 1                      | Master asserts                       |
| `arready`   | 1                      | Controller asserts when slot avail   |

### R (Read Data)

| Signal      | Width                   | Notes                                  |
|-------------|-------------------------|----------------------------------------|
| `rdata`     | `AXI_DATA_WIDTH`        | Beat data                              |
| `rid`       | `AXI_ID_WIDTH`          | Matches `arid`                         |
| `rresp`     | 2                       | `OKAY` (v1 always)                     |
| `rlast`     | 1                       | Final beat of burst                    |
| `rvalid`    | 1                       | Controller asserts when beat ready     |
| `rready`    | 1                       | Master asserts                         |

## Ordering Rules

- **Within an ID:** AXI4 ordering preserved. Reads from `arid = X` return in AR order; writes to `awid = X` commit in AW order.
- **Across IDs:** Out-of-order completion permitted by default (`AXI_OOO_ACROSS_IDS = true`). Disabling forces in-order across IDs at the cost of bandwidth.

## Burst Type Support

| Burst    | Mandatory | Notes                                                  |
|----------|-----------|--------------------------------------------------------|
| `FIXED`  | optional  | Enable with `SUPPORT_FIXED_BURST`; rarely useful for DRAM |
| `INCR`   | mandatory | Standard incrementing burst                            |
| `WRAP`   | optional  | Enable with `SUPPORT_WRAP_BURST`                       |

## Burst Length

AXI4 allows up to 256 beats. If a burst exceeds the remaining columns in its target row, `axi4_slave` splits internally; the SoC sees a single AW with the expected response time.

## Narrow Transfers

AXI4 supports `awsize < bus width`. The narrow case is handled via `wstrb` — the unused byte lanes are masked. The DRAM commits only the unmasked bytes.

## Posted Writes

B-channel responses are generated when the WR command is **scheduled**, not when wrdata commits. This is AXI4-compliant (`OKAY` indicates the transaction was accepted, not committed) and reduces master-side latency.

## Backpressure

- AW / AR `.ready` deassert when `txn_queue` is full
- W `.ready` deasserts when the write-data FIFO is full
- R / B `.valid` follow standard protocol; consumers may stall indefinitely without controller misbehavior

## AXI4 Reset

`axi_aresetn` is the AXI reset. When asserted, all AXI state is cleared. The internal controller pipeline is also reset (the two reset signals are tied together in v1 of this controller).
